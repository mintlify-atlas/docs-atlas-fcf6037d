The Puppet Firewall module lets you manage firewall rules as Puppet resources. Instead of writing iptables commands by hand, you declare what your firewall should look like — Puppet ensures it stays that way across every node, every run.

## What the module does

The module introduces two Puppet resource types:

- **`firewall`** — Manages individual iptables/ip6tables rules. Each rule is numbered for ordering and maps directly to an iptables rule on the system.
- **`firewallchain`** — Manages chains and their default policies. Use this to create custom chains or set the default policy on built-in ones like `INPUT` and `FORWARD`.

Rules take effect immediately when Puppet applies the catalog and persist across reboots automatically.

## Get started

<Steps>
  <Step title="Install the module">
    Install from the Puppet Forge using the `puppet module` tool:
    ```bash
    puppet module install puppetlabs-firewall
    ```
  </Step>
  <Step title="Create pre and post rule classes">
    Define `my_fw::pre` and `my_fw::post` classes to establish safe baseline rules before and after your custom rules run.
  </Step>
  <Step title="Declare your firewall rules">
    Use the `firewall` resource to declare rules in the Puppet DSL. Rules are applied in numeric order based on their title prefix.
  </Step>
  <Step title="Include the firewall class">
    Add `class { 'firewall': }` to your node to ensure required packages are installed and services are running.
  </Step>
</Steps>

## Supported platforms

The module supports these operating systems:

| Platform | Versions |
|---|---|
| RedHat / CentOS | 7, 8, 9 |
| Debian | 10, 11, 12 |
| Ubuntu | 18.04, 20.04, 22.04, 24.04 |
| SLES | 12, 15 |
| Rocky Linux | 8 |
| AlmaLinux | 8 |
| OracleLinux | 7 |

<Note>
Requires Puppet 8.x and the `puppetlabs/stdlib` module (>= 9.0.0).
</Note>

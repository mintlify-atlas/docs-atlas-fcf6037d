## Overview

The `firewall` resource type manages firewall rules on Linux systems via iptables. Each resource represents a single iptables rule and its position in a chain.

**Autorequires:**

- If Puppet is managing the iptables chains specified in the `chain` or `jump` parameters, the `firewall` resource will autorequire those `firewallchain` resources.
- If Puppet is managing the `iptables`, `iptables-persistent`, or `iptables-services` packages, the `firewall` resource will autorequire those packages to ensure that required binaries are installed.

## Providers

The `iptables` provider is the default and only supported provider for this resource type.

- **Required binaries:** `iptables-save`, `iptables`
- **Default for:** `kernel == linux`

## Examples

### Basic accept rule

```puppet
firewall { '100 allow http and https':
  dport => [80, 443],
  proto => 'tcp',
  jump  => 'accept',
}
```

### Drop rule

```puppet
firewall { '200 drop all other inbound traffic':
  chain  => 'INPUT',
  proto  => 'all',
  jump   => 'DROP',
}
```

### Stateful connection tracking

```puppet
firewall { '010 accept established and related connections':
  state => ['ESTABLISHED', 'RELATED'],
  jump  => 'accept',
}
```

### Port-based rule

```puppet
firewall { '022 allow ssh':
  dport   => 22,
  proto   => 'tcp',
  jump    => 'ACCEPT',
  source  => '10.0.0.0/8',
}
```

## Parameters

### `name`

**Type:** `Pattern[/(^\d+(?:[ \t-]\S+)+$)/]`

The canonical name of the rule. This name is also used for ordering, so you must prefix the rule with a number:

```puppet
firewall { '000 this runs first': ... }
firewall { '999 this runs last': ... }
```

Depending on the provider, the rule name is stored using the comment feature of the underlying firewall subsystem.

## Properties

### Core

<ParamField path="ensure" type="Enum[present, absent]" default="present">
  Whether this rule should be present or absent on the target system.
</ParamField>

<ParamField path="chain" type="String[1]" default="INPUT">
  Name of the chain the rule will be a part of. Ensure the chain you choose exists within your set table. Can be one of the built-in chains (`INPUT`, `FORWARD`, `OUTPUT`, `PREROUTING`, `POSTROUTING`) or a user-defined chain name.
</ParamField>

<ParamField path="table" type="Enum[nat, mangle, filter, raw, rawpost, broute, security]" default="filter">
  The table the rule will exist in. Valid options are `nat`, `mangle`, `filter`, `raw`, `rawpost`, `broute`, and `security`.
</ParamField>

<ParamField path="proto" type="Optional[Pattern[...]]" default="tcp">
  The specific protocol to match for this rule. Accepted values include `tcp`, `udp`, `icmp`, `esp`, `ah`, `vrrp`, `carp`, `igmp`, `ipv4`, `ospf`, `gre`, `cbt`, `sctp`, `pim`, `all`, and `ipencap`. Can be negated by prefixing with `! `.
</ParamField>

<ParamField path="protocol" type="Enum[iptables, ip6tables, IPv4, IPv6]" default="IPv4">
  The protocol used to set the rule. Allowed values are `iptables`, `ip6tables`, `IPv4`, and `IPv6`. Previously defined as `provider`, but renamed because the resource_api does not allow `provider` as an attribute title.
</ParamField>

<ParamField path="jump" type="Optional[String[1]]">
  The value for the iptables `--jump` parameter — the action to perform on a match. Common values are:

  - `ACCEPT` — the packet is accepted
  - `REJECT` — the packet is rejected with a suitable ICMP response
  - `DROP` — the packet is dropped

  Other valid values include `QUEUE`, `RETURN`, `DNAT`, `SNAT`, `LOG`, `NFLOG`, `NETMAP`, `MASQUERADE`, `REDIRECT`, `MARK`, and `CT`. Any valid chain name is also allowed.

  If you specify no value, the rule matches but performs no action.
</ParamField>

<ParamField path="goto" type="Optional[String[1]]">
  The value for the iptables `--goto` parameter. Similar to `jump`, but the rule does not return to the calling chain after reaching the end of the target chain. Accepts the same target values as `jump`.
</ParamField>

<ParamField path="log_tcp_sequence" type="Optional[Boolean]">
  When combined with `jump => 'LOG'`, enables logging of the TCP sequence numbers.
</ParamField>

<ParamField path="log_tcp_options" type="Optional[Boolean]">
  When combined with `jump => 'LOG'`, enables logging of the TCP packet header.

#### NFLOG parameters

<ParamField path="nflog_group" type="Optional[Integer[1, 65535]]">
  Used with `jump => 'NFLOG'`. The netlink group (0 – 65535) to which packets are sent (only applicable for `nfnetlink_log`). Defaults to 0.
</ParamField>

<ParamField path="nflog_prefix" type="Optional[String]">
  Used with `jump => 'NFLOG'`. A prefix string to include in the log message, up to 64 characters long, useful for distinguishing messages in the logs.
</ParamField>

<ParamField path="nflog_size" type="Optional[Integer[1]]">
  Used with `jump => 'NFLOG'`. The number of bytes to be copied to userspace (only applicable for `nfnetlink_log`). Overrides instance-level size settings.
</ParamField>

<ParamField path="nflog_threshold" type="Optional[Integer[1]]">
  Used with `jump => 'NFLOG'`. Number of packets to queue inside the kernel before sending them to userspace. Higher values reduce overhead per packet but increase delay. Defaults to 1.
</ParamField>

<ParamField path="nflog_range" type="Optional[Integer[1]]">
  Used with `jump => 'NFLOG'`. Deprecated — use `nflog_size` instead.
</ParamField>

### NAT

<ParamField path="tosource" type="Optional[String[1]]">
  When using `jump => 'SNAT'`, specifies the new source address. Accepts a single IP address or an inclusive range. Format: `[ipaddr[-ipaddr]][:port[-port]]`.
</ParamField>

<ParamField path="todest" type="Optional[String[1]]">
  When using `jump => 'DNAT'`, specifies the new destination address. Accepts a single IP address or an inclusive range with optional port. Format: `[ipaddr[-ipaddr]][:port[-port[/baseport]]]`.
</ParamField>

<ParamField path="toports" type="Optional[Variant[Integer[0, 65535], Pattern[...]]]">
  For `REDIRECT`/`MASQUERADE`, the port that replaces the destination/source port. Accepts a single port or an inclusive range.
</ParamField>

<ParamField path="to" type="Optional[String[1]]">
  For `NETMAP`, replaces the destination IP.
</ParamField>

<ParamField path="random" type="Optional[Boolean]">
  When using a jump value of `MASQUERADE`, `DNAT`, `REDIRECT`, or `SNAT`, enables randomized port mapping.
</ParamField>

<ParamField path="random_fully" type="Optional[Boolean]">
  When using a jump value of `MASQUERADE`, `DNAT`, `REDIRECT`, or `SNAT`, enables fully randomized port mapping.
</ParamField>

<ParamField path="gateway" type="Optional[Pattern[...]]">
  The TEE target clones a packet and redirects the clone to another machine on the local network segment. `gateway` is the target host's IP address.
</ParamField>

### Rate limiting

<ParamField path="limit" type="Optional[Pattern[...]]">
  Rate limiting value for matched packets. Format: `rate/[second|minute|hour|day]`.

  Examples: `'50/sec'`, `'40/min'`, `'30/hour'`, `'10/day'`.
</ParamField>

<ParamField path="burst" type="Optional[Integer[1]]">
  Rate limiting burst value (per second) before limit checks apply.
</ParamField>

#### Hashlimit parameters

<ParamField path="hashlimit_name" type="Optional[String[1]]">
  The name for the `/proc/net/ipt_hashlimit/foo` entry. Required when setting any other hashlimit values (along with either `hashlimit_upto` or `hashlimit_above`).
</ParamField>

<ParamField path="hashlimit_upto" type="Optional[Pattern[...]]">
  Match if the rate is below or equal to the specified amount. Required (with `hashlimit_name`) when setting any other hashlimit values. Allowed forms: `'40'`, `'40/sec'`, `'40/min'`, `'40/hour'`, `'40/day'`.
</ParamField>

<ParamField path="hashlimit_above" type="Optional[Pattern[...]]">

### Time matching

<ParamField path="time_start" type="Optional[Pattern[...]]">
  Only match during the given daytime. Possible time range is `00:00:00` to `23:59:59`. Leading zeroes are allowed and interpreted as base-10.

<ParamField path="time_stop" type="Optional[Pattern[...]]">
  Only match until the given daytime. Possible time range is `00:00:00` to `23:59:59`. Leading zeroes are allowed and interpreted as base-10.

<ParamField path="date_start" type="Optional[Pattern[...]]">
  Only match during a given date range. Must be in ISO 8601 `T` notation. Possible range: `1970-01-01T00:00:00` to `2038-01-19T04:17:07`.

### Recent module

<ParamField path="recent" type="Optional[Enum[set, update, rcheck, remove, ...]]">
  Enable the recent module. Takes one of `set`, `update`, `rcheck`, or `remove` as an argument. Can be negated with `! `.

  ```puppet
  # Drop traffic from badguys seen in the last 60 seconds
  firewall { '100 Drop badguy traffic':
    recent   => 'update',
    rseconds => 60,
    rsource  => true,
    rname    => 'badguy',
    jump     => 'DROP',
    chain    => 'FORWARD',
  }
  ```
</ParamField>

<ParamField path="rname" type="Optional[String[1]]">
  Recent module — the name of the list. Defaults to `DEFAULT` when `recent` is set.
</ParamField>

<ParamField path="rsource" type="Optional[Boolean]">
  Recent module — add the source IP address to the list. Mutually exclusive with `rdest`. Defaults to `true` when `recent` is set.
</ParamField>

<ParamField path="rdest" type="Optional[Boolean]">
  Recent module — add the destination IP address to the list. Mutually exclusive with `rsource`. Must be boolean `true`.
</ParamField>

<ParamField path="rseconds" type="Optional[Integer[1]]">
  Recent module — narrows the match to only happen when the address is in the list and was seen within the last given number of seconds. Used with `recent => 'rcheck'` or `recent => 'update'`.
</ParamField>

<ParamField path="rhitcount" type="Optional[Integer[1]]">
  Recent module — narrows the match to only happen when the address is in the list and packets have been received greater than or equal to the given value. Used with `recent => 'update'` or `recent => 'rcheck'`.
</ParamField>

<ParamField path="rttl" type="Optional[Boolean]">
  Recent module — narrows the match to only happen when the address is in the list and the TTL of the current packet matches the packet which hit `recent => 'set'`. Must be boolean `true`.
</ParamField>

<ParamField path="reap" type="Optional[Boolean]">
  Recent module — can only be used with `rseconds`. When used, causes entries older than `rseconds` to be purged. Must be boolean `true`.
</ParamField>

<ParamField path="mask" type="Optional[Pattern[...]]">
  Recent module — sets the mask to use when `recent` is enabled. Defaults to `255.255.255.255` when `recent` is set.
</ParamField>
